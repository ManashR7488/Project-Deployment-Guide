üöÄ Launch an EC2 Instance

Steps to Launch Your Instance

  1. Create an EC2 instance (Ubuntu 22.04 LTS)
  2. Create or select a Key Pair (.pem file)
  3. Create a Security Group allowing SSH (22), HTTP (80), HTTPS (443), and custom ports

Security Group Configuration

Configure security group inbound rules 
# Configure security group inbound rules
# Type: SSH, Protocol: TCP, Port: 22, Source: Your IP
# Type: HTTP, Protocol: TCP, Port: 80, Source: 0.0.0.0/0
# Type: Custom TCP, Protocol: TCP, Port: 3000, Source: 0.0.0.0/0

‚ö†Ô∏è Security Note: Principle of least privilege: open only the ports your application absolutely needs.

üîí Secure Your SSH Key

Windows (PowerShell)

Navigate to keys directory 
cd C:\path\to\keys
Remove inheritance 
icacls "Key-path.pem" /inheritance:r
Grant read access to current user 
icacls "Key-path.pem" /grant:r $env:USERNAME`:R

macOS/Linux (Terminal)

Set proper permissions 
chmod 400 key.pem

Connect to Your EC2 Instance

SSH connection command 
ssh -i "Key-path.pem" ubuntu@<EC2_PUBLIC_DNS>

üì• Clone Your Repository

Public Repository

Clone public repository 
git clone <clone-URL>

Private Repository

For private repositories, you'll need to set up SSH authentication:

Generate SSH key 
ssh-keygen -t ed25519 -C "your_email@example.com"
Display public key 
cat ~/.ssh/id_ed25519.pub
Test GitHub connection 
ssh -T git@github.com
Clone private repository 
git clone git@github.com:username/repo.git

üóëÔ∏è Kill Processes on a Port

Sometimes you need to free up a port by killing running processes:

List processes on port 3000 
sudo lsof -i :3000
Kill specific process by PID 
sudo kill -9 <PID>
Kill all processes on port 3000 (one-liner) 
sudo kill -9 $(sudo lsof -t -i:3000)

‚öôÔ∏è Manage Your App with PM2

üîê Enable HTTPS with NGINX & Certbot

Install and Configure NGINX

Install NGINX 
sudo apt install -y nginx
Allow NGINX through firewall 
sudo ufw allow 'Nginx Full'
Remove default configuration 
sudo rm /etc/nginx/sites-enabled/default
Create new site configuration 
sudo nano /etc/nginx/sites-available/myapp

NGINX Configuration

Add this configuration to your NGINX site file:

NGINX site configuration 
server {
    listen 80;
    server_name yourdomain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}

SSL/TLS with Let's Encrypt

Install Certbot 
sudo apt install -y certbot python3-certbot-nginx
Obtain SSL certificate 
sudo certbot --nginx -d yourdomain.com
Test auto-renewal 
sudo certbot renew --dry-run

‚öôÔ∏è CI/CD Deployment with GitHub Actions

Generate SSH Key for Deployment

Generate RSA key for deployment 
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

Required GitHub Secrets

Add these secrets to your GitHub repository settings (Settings ‚Üí Secrets and variables ‚Üí Actions):

EC2_SSH_KEY

Contents of ec2-deploy-key (private key)

EC2_USER

ubuntu

EC2_HOST

Your EC2 instance public IP

TARGET_DIR

/home/ubuntu/code/projectRepo

GitHub Actions Workflow

Create .github/workflows/deploy.yml in your repository:

GitHub Actions workflow file 
name: Deploy to EC2

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout Code
        uses: actions/checkout@v3
      - name: Deploy to EC2
        run: |
          ssh -i ~/.ssh/ec2-deploy-key ${{ secrets.EC2_USER }}@${{ secrets.EC2_HOST }} << 'EOF'
            cd ${{ secrets.TARGET_DIR }}
            git pull origin main
            npm install
            pm2 restart all || pm2 start index.js --name my-app
          EOF

üõ†Ô∏è Troubleshooting

Issue Resolution
SSH connection refused Ensure port 22 is open in both AWS security group and UFW, and your IP is whitelisted
Permission denied (SSH key) File mode must be 400 (Linux/macOS) or restricted via icacls (Windows)
App not running after reboot Execute pm2 startup then pm2 save so processes respawn automatically
NGINX configuration errors Run sudo nginx -t to validate, fix errors, then sudo systemctl reload nginx
Let's Encrypt fails Confirm your domain's A-record points to the EC2 IP and ports 80/443 are open
GitHub Actions cannot connect Check EC2_SSH_KEY formatting, correct EC2_HOST, and ensure port 22 is open to GitHub ranges
Port appears closed externally Verify both the AWS security group and sudo ufw status include the required port
Node.js version conflicts Use nvm to manage Node.js versions: curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
